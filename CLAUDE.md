# Card Hero - 프로젝트 가이드

## 프로젝트 개요

- **엔진**: Godot
- **장르**: 카드 + 전투 RPG / 2D 탑뷰 액션
- **핵심 재미**: 다양한 빌드 조합, 덱빌딩, 수집
- **수집/강화 루프**: 조합으로 카드 제작 → 점진적으로 강해지는 적 → 처치 시 소재 획득 → 더 강한 카드 제작 (과금 없이 노가다 기반)

---

## 스토리

강했던 용사 파티가 어떤 존재에 의해 힘을 잃고 카드를 통해 힘을 되찾아가는 여정

## 속성 시스템

### 속성 상성
불 → 풀 → 땅 → 전기 → 물 → 불

### 속성별 색상
| 속성 | 색상 |
|------|------|
| 🔥 불 | 빨강 |
| 🌿 풀 | 초록 |
| 💧 물 | 파랑 |
| 🌍 땅 | 회색 |
| ⚡ 전기 | 노랑 |

---

## 전투 시스템

### 기본공격 / 특수공격 / 회피

| 직업 | 기본공격 | 특수공격 | 회피 |
|------|----------|----------|------|
| 전사 | 베기 | 차지공격 | 대쉬 (짧고 무적시간 적음) |
| 탱커 | 방패치기 | 방패들기 | ❌ |
| 유틸 | ❌ | ❌ | 구르기 (무적 길고, 이동거리 김) |
| 법사 | 완드 휘두르기 | 차징 | 점멸 (쿨타임 있음, 거리 짧음) |
| 사제 | 힐 (쿨타임 김) | 이속버프 (쿨타임 김) | ❌ |

### 코스트 / 드로우 시스템

| 항목 | 내용 |
|------|------|
| 최대 코스트 | 5 |
| 코스트 충전 | 기본공격 or 시간에 따라 충전 |
| 드로우 | 특수공격 or 카드 효과 |
| 손패 | 최대 5장, 넘치면 뽑은 카드 버리기 |
| 카드 분류 | 무기카드 / 패시브 / 액티브 |
| 덱 구성 | 15장, 등급마다 들어갈 수 있는 최대치 다름 |

### 직업별 코스트/드로우 획득 방식
- **전사**: 공격이 적에게 피격 시 코스트 획득, 특수공격이 적에게 피격 시 드로우
- **탱커**: 회피 불가 / 완벽한 타이밍 방어 시 코스트 획득
- **유틸**: 무기 없이 시작 / 회피 시 코스트 획득
- **법사**: 차징 or 근거리 공격 시 코스트 획득
- **사제**: 코스트가 자동으로 회복

## 카드 시스템

### 카드 유형
- 무기카드 / 패시브 / 액티브 / 타겟팅 / 즉발
- 효과 종류: 공격 스킬 / 버프 / 디버프 / 지형지물 / 증강 / 드로우 / 특수효과 / 치유

### 카드 레어도
일반 / 레어 / 유니크 / 전설

### 카드 데이터 구조 (직업별 DB)
- 카드이름, 카드유형 (스킬/무기/파워), 레어도, 코스트, 카드설명, 비고
- 유틸 카드에는 "기본 지급" 여부 필드 있음

### 카드 제작
- 카드 제작 레시피 유사도 기반 (재료 조합)

## 프로젝트 구조

```
card-hero/
├── scenes/
│   ├── player/       # 캐릭터별 씬
│   ├── enemies/      # 몬스터 씬
│   ├── ui/card/      # 카드 UI 씬
│   └── dungeon/      # 스테이지/던전 씬
├── scripts/
│   ├── player/       # 캐릭터 스크립트
│   ├── enemies/      # 몬스터 AI
│   ├── cards/        # 카드 로직
│   ├── managers/     # 게임 매니저 (덱, 전투, 스테이지 등)
│   └── components/   # 재사용 컴포넌트
└── assets/
	├── sprites/      # 스프라이트 이미지
	└── audio/        # 효과음/배경음
```
## 네이밍 컨벤션
- C# 클래스: PascalCase
- 변수: camelCase
- 씬 파일: snake_case.tscn
- 스크립트: PascalCase.cs

## Autoload 싱글톤
- GameManager: 전체 게임 상태, 씬 전환
- CardManager: 덱/핸드 관리
- DungeonManager: 룸 생성/이동
